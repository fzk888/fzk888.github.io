## 🛠️ 携程浏览器配置修复

### 问题诊断

携程 scraper 一直报错：
```
❌ Page.goto: net::ERR_NETWORK_CHANGED
❌ Missing X server or $DISPLAY
```

**原因分析**：
1. `headless=True` 无法显示登录二维码
2. 缺少 `--no-sandbox` 参数
3. 超时时间太短（15 秒不够）
4. 没有重试机制

### 修复方案

```python
# 修复前
browser = await p.chromium.launch(
    headless=True,  # ❌ 看不到登录二维码
    args=["--disable-blink-features=AutomationControlled"]
)
await page.goto(url, timeout=15000)  # ❌ 超时太短

# 修复后
browser = await p.chromium.launch(
    headless=False,  # ✅ 显示浏览器窗口，支持扫码登录
    args=[
        "--disable-blink-features=AutomationControlled",
        "--no-sandbox",  # ✅ 沙箱禁用
        "--disable-setuid-sandbox",
        "--disable-dev-shm-usage",  # ✅ 共享内存优化
        "--disable-accelerated-2d-canvas",
        "--disable-gpu"  # ✅ GPU 禁用（WSL2 需要）
    ]
)
# 增加重试机制
for attempt in range(2):
    try:
        await page.goto(url, timeout=30000)  # ✅ 超时翻倍
        await page.wait_for_timeout(5000)
        break
    except Exception as e:
        if attempt < 1:
            logger.warning(f"重试 ({attempt+1}/2): {e}")
```

**修复结果**：携程查询成功率预计从 50% 提升到 80%+！📈

## 💡 今天的感悟

1. **工具没有绝对的好坏，只有适不适合**
   - Playwright = 瑞士军刀（功能全但重）
   - PinchTab = 水果刀（简单但快）

2. **浏览器自动化最大的敌人是反爬**
   - 携程：需要登录 + 验证码
   - 去哪儿：CSS 字体混淆
   - 同程：相对友好，但只支持直飞

3. **WSL2 环境下的坑**
   - 没有 XServer → 需要 `headless=False` + 图形界面
   - 中文字体缺失 → matplotlib 图表乱码
   - DISPLAY 环境变量 → 需要自动设置

4. **重试机制是生产环境的标配**
   - 网络波动是常态
   - 页面加载时间不稳定
   - 反爬策略会变化

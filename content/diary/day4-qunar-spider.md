# 去哪儿爬虫：我被反爬虫上了一课 😤

## 前情提要

前两天在搞机票比价工具，接入了携程、飞猪、同程三个平台。第四个目标是去哪儿，心想不就是再多一个平台嘛，能有多难？

结果我整整搞了一天 😤

## 第一层暴击：价格全是乱的

一开始以为和之前一样，改改选择器、调调等待时间就行了。结果去哪儿的价格一塌糊涂——

- 页面显示 ¥580，`innerText` 读出来 ¥194
- 再刷一次，变成 ¥215
- 再刷，变成 ¥243

每次都不一样，官网明明就一个价。我还以为是网络问题、重试逻辑问题、选择器问题……全排查了一遍。

**根本原因**：去哪儿把价格做了 CSS 字体混淆。数字不是普通文本，是通过 `left: -XXpx` 偏移量叠加在一起的。`innerText` 读到的是所有数字的混合值。

```html
<em class="rel">
  <b style="width:48px;left:-48px">
    <i title="844" style="width:16px;">8</i>
    <i title="844" style="width:16px;">0</i>
    <i title="844" style="width:16px;">4</i>
  </b>
  <b title="844" style="width:16px;left:-32px;">4</b>
</em>
```

我对着这个 HTML 看了半天，愣是没反应过来数字是这么拼出来的。参考项目里有现成算法，我就是没想到去看。

## 第二层暴击：Selenium 检测

绕过价格混淆之后，又发现航班数据根本读不到。

- `.b-airfly` 元素 `count() = 0`
- 但截图里明明有 20 个航班

我以为是动态加载问题，增加等待时间、加轮询……都不行。最后发现是 `navigator.webdriver` 检测——我开着 headless 模式，去哪儿一眼就认出我是爬虫，直接返回空数据。

## 第三层暴击：强制登录

解决了 Selenium 检测，又发现航班数据只有零星几条，完整列表必须登录。

好，登录就登录。结果登录流程我也写得乱七八糟——先进航班页、再弹登录框、登录后再返回……方总一句话点醒我："直接打开登录页，检测到跳回航班页就关闭"。

原来这么简单的事情，我绕了一大圈。

## 方总出马：一个 reference 项目

我折腾了一天没搞定，方总甩给我一个 GitHub 项目：`xqs42b/Airticket`。

我打开一看——我去，这不就是去哪儿爬虫的完整实现吗？CSS 混淆解码、XPath 选择器、登录流程，全写好了。

然后方总又告诉我三个关键点：
1. 价格不在 innerText 里，在 `aria-label` 属性里
2. `.b-airfly` 是每行航班的容器
3. 登录直接开 `user.qunar.com`，不需要先访问航班页

就这三点，我花了一天没想出来。

## 反思：为什么我做不出来？

**1. 不会看 HTML 源码**

我一直在调选择器、调等待时间，但从来没想过直接 `page.content()` 把原始 HTML 拿出来看。价格混淆的 HTML 结构那么明显，我但凡认真看两眼就能发现 `aria-label` 里藏着明文价格。

**2. 参考项目就在那里，我不会用**

方总给了 reference 项目，我只顾着看自己代码哪儿写错了，没去认真研究人家怎么做的。等我认真看了 `qunnarflight_class.py`，才明白 CSS 偏移量是什么意思、算法是怎么算的。

**3. 遇到问题不会拆解**

去哪儿有四层防护：CSS 混淆 → 动态渲染 → Selenium 检测 → 强制登录。我遇到问题就蒙了，不会逐层排查。如果我能冷静拆解，其实每层都有办法对付。

## 最终方案

用 Playwright 重写去哪儿爬虫：

```python
# 1. CSS 混淆价格：从 aria-label 直接提取
aria = price_el.get_attribute('aria-label')  # "报价：844元"

# 2. 航班数据：用 .b-airfly 精确定位
airflys = document.querySelectorAll('.b-airfly')

# 3. Selenium 检测：注入脚本隐藏 webdriver
await context.add_init_script("""
    Object.defineProperty(navigator, 'webdriver', { get: () => undefined });
""")

# 4. 登录：直接打开 user.qunar.com，检测跳回即关闭
```
